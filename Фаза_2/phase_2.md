Решая задачу (например, в AMPL / NEOS / Pyomo), получаем следующее распределение задач между участниками:

| Задача | Ответственный (R) | Обоснование |
|--------|-------------------|-------------|
| **T1** Анализ продукта                | **M (Мирослав)** | Высокое предпочтение к аналитике, средняя сложность |
| **T2** Определение акторов            | **M (Мирослав)** | Сильный интерес к моделированию ролей |
| **T3** Набросок Use Cases             | **N (Никита)**   | Максимальное предпочтение к UC, хорошее чувство сценариев |
| **T4** Детализация Use Cases          | **M (Мирослав)** | Много текстовой работы и альтернативных сценариев |
| **T5** Подготовка к COSMIC            | **E (Евгений)**  | Высокое предпочтение к формальным моделям и расчётам |
| **T6** Подготовка к UCP               | **E (Евгений)**  | Аналогично, работа с методикой оценивания |

| **T8** Расчёт Use Case Points         | **N (Никита)**   | Интерес к практическим оценкам трудоёмкости |

### 6.1. Баланс нагрузки (по сложности)

- Евгений: **T5 + T6 + T7 = 5 + 5 + 8 = 18 SP**
- Мирослав: **T1 + T2 + T4 + T10 = 5 + 3 + 8 + 4 = 20 SP**
- Никита: **T3 + T8 + T9 = 5 + 7 + 4 = 16 SP**

Нагрузка распределена **достаточно равномерно**, при этом:
- Евгений фокусируется на численных методах (COSMIC / UCP).  
- Мирослав — на аналитике и документации.  
- Никита — на Use Cases и экспертных оценках.
